sonar学习笔记

2016-12-06

sonar简介

sonar是一款强大的代码质量管理软件。支持本地和服务端分析

sonarQube

sonarQube是sonar的服务端。使用sonarQube可以自己假设sonar的服务端，并通过网站访问管理。

下载

官网地址：http://www.sonarqube.org/
ps:

  1. 以前的版本是sonarQube和sonarRunner，现在已不支持。qube至少要5.X版本，最好使用最新版本
  2. sonarQube和scanner版本必须匹配，下载时可以看到对应的版本需求

mysql数据库配置

创建sonar数据库和用户
语句：
CREATE DATABASE sonar CHARACTER SET utf8 COLLATE utf8_general_ci;
CREATE USER 'sonar' IDENTIFIED BY 'sonar';
GRANT ALL ON sonar.* TO 'sonar'@'%' IDENTIFIED BY 'sonar';
GRANT ALL ON sonar.* TO 'sonar'@'localhost' IDENTIFIED BY 'sonar';
FLUSH PRIVILEGES;
查看运行结果：


ps：mysql至少需要5.6+版本，低版本不支持

配置sonarQube

配置文件为qube安装目录下的conf文件下的sonar.properties文件
主要配置内容：数据库地址、账号密码和数据库驱动
sonar.jdbc.username=数据库用户
sonar.jdbc.password=密码
sonar.jdbc.url=jdbc:mysql://localhost:3307/sonar? useUnicode=true&characterEncoding=utf8&rewriteBatchedStatements=true&useConfigs=maxPerformance
sonar.jdbc.driver:com.mysql.jdbc.Driver
ps:

  1. 数据地址后的四个设置必须带上，这是sonarQube的前置需求
  2. 数据库驱动只需要配置即可，sonarQube已经集成的大部分驱动，除了oracle。使用oracle数据库需要下载驱动到指定目录下
  3. 网上教程需要下载mysql驱动是因为版本低的原因（网上教程基本都是低版本的，只能作为参考）

    接下来可以使用sonarScanner分析代码并上传

    sonarScanner

    基础配置

    配置scanner

    配置文件为conf目录下的sonar-scanner.properties
    配置项：
    sonar.host.url=sonar服务端地址
    sonar.sourceEncoding=UTF-8
    sonar.login=admin（登录名）
    sonar.password=admin（登录密码）

    配置系统参数

    右击计算机-属性-高级系统设置-环境变量
    新建SONAR_SCANNER_HOME，值为scanner解压目录
    在path值后加上%SONAR_SCANNER_HOME%\bin;

    检查系统参数是否配置成功

    cmd命令窗口运行sonar-scanner -v命令，出现如下窗口表示配置成功。出错请检查配置及参数设置

    配置项目

    在要分析的项目根目录创建sonar-project.properties文件
    配置项：
    sonar.projectKey=项目名
    sonar.projectName=项目名
    sonar.projectVersion=版本号
    sonar.sources=源文件夹路径（多个用逗号隔开）
    sonar.sourceEncoding=UTF-8
    binaries=WebRoot/WEB-INF/classes

    分析代码

    打开cmd窗口
    切换到分析项目的目录
    使用到的命令：
  8. d: –切换根目录
  9. cd 文件夹名 –进入指定目录
  10. cd .. –返回上级目录

    分析失败

    Unsupported major.minor version 52.0

    原因：java版本问题，默认的java版本和sonar不匹配
    修改方法：
  11. cmd命令执行java -version查看当前版本信息
  12. 修改系统参数的JAVA_HOME和path（最好系统和用户的都修改）,cmd命令执行set path
    ps:java默认版本为path里的第一个jdk对应的版本
  13. 重新打开cmd窗口执行java -version命令查看默认版本
    ps:必须重新打开cmd窗口才能看到最新的版本

Fail to insert data of CE task..

图示：

原因：MySQL数据库的max_allowed_packet值太小了，不能一次插入分析的内容数据
修改方法：

  1. mysql运行show VARIABLES like ‘%max_allowed_packet%’;命令查看当前的设置
  2. 修改mysql的配置文件
    在[mysqld]下加上max_allowed_packet = 32M
  3. 重启mysql服务，运行命令查看是否设置成功；成功后再重启sonarQube服务

安装两个mysql

原因：目前大部分计算机使用的都是MySQL5.5，sonar新版本不支持该版本。同时又不希望直接删掉5.5来安装更高的版本
方法：

  1. 下载5.7版本的MySQL数据库，解压到与之前版本不同的目录
  2. 新增my.ini文件，自定义配置项
    端口号和地址都不能和旧的相同
  3. 在解压目录下新增对应的data文件夹
  4. cmd窗口切换到解压目录的bin文件下
  5. 运行mysqld install MySql3307 –defaults-file=”D:\mysql-5.7.16-winx64\my.ini”命令
    MySql3307为新服务的名称，不能和旧的相同。后面的地址为使用的配置文件地址
  6. 修改注册表
    6.1 打开regedit程序
    6.2 HKEY_LOCAL_MACHINE–>SYSTEM–>CurrentControlSet–>Services下找到MySql3307
    6.3 修改ImagePath

    6.4 重启电脑或打开任务管理器结束explorer.exe,再应用程序-新任务打开explorer.exe
  7. 启动服务
    7.1 右键计算机-管理-服务和应用程序-服务，找到MySql3307。查看属性，文件路径是否正确
    7.2 启动服务
    7.3 启动失败可能原因
    7.3.1 文件路径不对，注册表修改后必须重启才有效，方法参照步骤6.4
    7.3.2 数据库配置错误，初始时只要保留基本配置即可

    sonarLint

    插件安装

    使用eclipse自带的Marketplace安装
    ps:安装前最好先卸载掉之前的lint和qube插件
    方法：help-Installing Details

    配置

    安装成功后打开SonarQube Servers、SonarLint Issues窗口

    点击SonarQube Servers下的Connect to a SonarQube server

    输入sonar server的相关信息，确保连接成功

    右键-Update all..可以同步更新服务端配置到本地

    分析项目

    选择项目->sonar-lint->Bind to a SonarProject

    若之前没有配置SonarProject,则红框内是新增的链接
    SonarQube Project里可以搜索与服务端匹配的项目，最后点击finish

项目图标变为如下情况表示成功

ps:

  1. 每个项目最好匹配同名的服务端项目，避免冲突
  2. sonarlint console可以查看执行过程，点击console下拉窗口可以看见

    分析时报错

    打开文件自动分析代码时，IO错误


    产生过程：
    关联服务端project后取消关联，再次关联分析代码时
    错误原因：
    再次关联时配置没有同步下来
    解决方法：
    servers-update all project bildings
    ps:
    update配置时可能报如下错误，更新失败。重启eclipse即可

    查看分析规则详情

    sonarLint Issues窗口-右键问题-rule description后在SonarLint Rule Description窗口可以看到详情
